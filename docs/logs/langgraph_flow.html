Tool Registry — 9 tools + dispatch map

💰 price

Live price for stocks, forex, crypto, indices via yfinance fast_info.

param: ticker (e.g. AAPL, RELIANCE.NS, BTC-USD, EURUSD=X)
Fallback: tries ticker + ".NS" if bare uppercase. Returns error string on failure.

📊 indicators

Fetches specific technical indicators. Tries Supabase cache first, falls back to yfinance 2y history. Patches missing fields from yfinance if cache is partial.

params: ticker, indicators (comma-list or "all")
Batch mode: ThreadPoolExecutor(max_workers=8) for multi-ticker. Cache miss → yfinance. Partial cache → hybrid patch.

🔍 analyze

Full technical + fundamental narrative. Extracts ticker via regex fast-path or LLM fallback. Fetches 3mo/6mo history for SMA/RSI/52w range.

param: query (natural language)
Batch: defers to indicators data already in scratchpad. History empty → partial data_context. Always appends [ANALYSIS COMPLETE] marker.

💱 forex

Analyzes currency pair via analyze_forex_pair(). Returns rate, change%, trend, RSI, SMA20/50, signals.

param: pair (e.g. EUR/USD, USD/INR)
Returns error string if analyze_forex_pair() raises or returns {"error": ...}.

📋 options

Options chain or single contract. Detects contract symbol (len>10 + C/P) vs underlying. Returns Greeks, IV, near-the-money strikes.

param: symbol (AAPL or AAPL240315C150)
get_options_chain() returns None → error string. Contract not found → error string.

🛢️ futures

Commodity spot (gold/silver/oil/etc.) or futures curve (contango/backwardation). Keyword map → commodity symbol. Regex extracts futures symbol.

param: query (natural language)
Commodity spot fails → falls through to futures curve. Defaults to /ES if no symbol found.

📈 backtest

Runs run_strategy_backtest(). Caches full result in _backtest_full_result_cache["last"]. Returns metrics summary string.

param: strategy (natural language)
Exception → error string. _build_result() pops cache to return full structured object.

🔎 screen

Runs run_stock_screener(). Caches full result in _screen_full_result_cache["last"]. Returns top 10 matches summary.

param: query (natural language criteria)
Exception → error string. _build_result() pops cache for structured response.

💬 general

Answers finance/trading questions from LLM knowledge. No live data. Calls generate_response() directly.

param: question
Exception → error string. Used when no market data is needed.
Data Source Priority (indicators tool)
1. Supabase cache (load_technicals_from_supabase) — fastest, pre-computed
2. If cache hit but fields missing → yfinance 2y history to patch gaps
3. If cache miss → yfinance 2y → 1y → 6mo (progressive fallback)
4. ATR14 always computed from raw history (not in Supabase)
5. All data fetches traced via @traceable (Langfuse observability)

Step-by-Step Sequence — "Analyze RELIANCE.NS" example
1
Caller → run_research_agent("Analyze RELIANCE.NS")

Builds initial_state. Calls _react_graph.invoke(initial_state). LangGraph starts at entry point "reason".

2
reason node (step 0)

_build_react_prompt() assembles: tool schema + conversation history (last 4 turns) + empty scratchpad + asset hint ("Detected: equity | RELIANCE.NS"). Calls LLM.

3
LLM responds → tool_call: indicators

_parse_react_response() extracts {"action":"tool_call","tool":"indicators","tool_input":{"ticker":"RELIANCE.NS","indicators":"rsi,macd,sma50,sma200,bb_upper,bb_lower"}}. Appended to scratchpad with empty observation.

4
_route() → execute_tool

scratchpad[-1].observation == "" → routes to execute_tool node.

5
execute_tool node — indicators

No duplicate. Calls _execute_indicators("RELIANCE.NS", "rsi,macd,..."). Tries Supabase cache → hit. Checks for missing fields → none missing. Formats indicator table. Returns observation string. step_count → 1.

6
_route() → reason (no done-marker)

Observation doesn't contain [ANALYSIS COMPLETE]. Routes back to reason.

7
reason node (step 1)

Prompt now includes indicators observation in scratchpad. LLM decides to call analyze with the indicator data.

8
execute_tool node — analyze

_execute_analyze("Analyze RELIANCE.NS"). Fast-path regex finds "RELIANCE.NS". Fetches fast_info (price, prev_close). Fetches 3mo history. Computes SMA20/50, RSI, 52w range. Calls LLM with data_context. Appends [ANALYSIS COMPLETE -- call final_answer now]. step_count → 2.

9
Done-marker detected in execute_tool

Observation contains "[ANALYSIS COMPLETE". Collects all observations (indicators + analysis). Builds synthesis prompt. Calls LLM → final_answer. Sets state.final_answer. Returns to _route().

10
_route() → END

state.final_answer is non-empty → returns END.

11
_build_result(final_state)

tools_used = ["indicators", "analyze"]. dominant_type = "analysis". Applies _fix_table_formatting(). Returns {"type":"analysis","answer":"...markdown...","steps":[...],"step_count":2}.

Streaming variant differences
S1
run_research_agent_stream() called

Creates queue.Queue(). Injects into initial_state as _stream_queue. Starts daemon thread via copy_context().run(_worker_body) (preserves Langfuse context vars).

S2
reason node emits step events

Each time reason picks a tool call, it puts {"event":"step","thought":...,"tool":...,"label":"Computing technical indicators","step":N} on the queue. Main thread yields data: {...}\n\n SSE immediately.

S3
Worker puts result or error

On success: {"event":"result","type":...,"answer":...,"steps":...}. On exception: {"event":"error","message":...}. Then puts SENTINEL object.

S4
Main generator exits on SENTINEL

while True: item = q.get(); if item is SENTINEL: break; yield f"data: {item}\n\n"

Key design decisions worth mentioning in interview
1. Graceful degradation chain: LangGraph unavailable → backup module → never crashes caller
2. Done-marker pattern: Tools signal completion via string markers, avoiding extra LLM round-trips
3. Duplicate-call guard: Prevents infinite loops when LLM repeats the same tool call
4. Hybrid data sourcing: Supabase cache → yfinance patch → full yfinance (cost/latency optimized)
5. Streaming via queue + thread: Decouples LangGraph's synchronous invoke from SSE generator
6. copy_context(): Preserves Langfuse trace context vars across thread boundary
7. MAX_STEPS=10 hard cap: Always produces an answer — never hangs indefinitely
8. Multi-stage JSON parser: Handles LLM formatting inconsistencies without crashing
